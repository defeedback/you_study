# 岗位方向分类

本文件将 30 份岗位从“原始批次顺序”重组为“学习和求职方向”，用于判断 AI Agent 开发需要优先补哪些能力。

## 分类总览

| 方向 | 代表岗位 | 适合目标 |
|---|---|---|
| AI Agent 应用开发 | JD-01、JD-02、JD-05、JD-12、JD-14、JD-15、JD-16、JD-17、JD-20、JD-21、JD-22、JD-25、JD-29、JD-30 | 想做大模型应用、智能体、工具调用、业务 Agent 的开发者 |
| RAG / 知识库 / 工作流工程 | JD-06、JD-13、JD-19、JD-26 | 想做知识库问答、企业文档检索、智能工作流的开发者 |
| AI 应用后端开发 | JD-06、JD-08、JD-13、JD-22、JD-23、JD-28 | 有后端基础，希望转向 LLM / Agent 应用工程的人 |
| 模型训练 / 微调 / 算法工程 | JD-03、JD-07、JD-11、JD-18 | 偏算法、模型部署、微调、深度学习工程方向的人 |
| AI 产品 / 运营 / 业务落地 | JD-04、JD-10、JD-24 | 偏产品、运营、流程设计、业务落地和 AI 工具应用的人 |
| 行业场景型 AI 应用 | JD-05、JD-14、JD-20、JD-24、JD-27、JD-29、JD-30 | 想把 Agent 落地到教育、餐饮、办公、游戏、新零售等具体业务的人 |

## 1. AI Agent 应用开发

### 方向说明

这是 30 份 JD 中最核心的方向，重点是把 LLM 变成能完成任务的应用系统，而不只是聊天机器人。

### 典型岗位

- JD-01 AI工程师
- JD-02 AI应用工程师
- JD-05 Agent开发工程师（教育方向）
- JD-12 AI应用工程师
- JD-14 AI应用研发工程师（餐饮业务）
- JD-15 AI应用开发工程师（校招·2026届）
- JD-16 AI Agent开发工程师
- JD-17 AI应用开发工程师
- JD-20 AI agent工程师（办公场景）
- JD-21 AI应用开发工程师（盯盯拍）
- JD-22 AI Agent开发工程师
- JD-25 AI Agent开发工程师（静距离）
- JD-29 AI应用研发工程师（喜茶）
- JD-30 Agent开发工程师（德特赛维）

### 核心能力

- LLM API 调用
- Prompt Engineering
- Function Calling / Tool Calling
- Agent 任务规划
- 记忆管理
- 多轮对话
- 多智能体协作
- 工作流编排
- 业务 API / 外部工具集成

### 学习重点

1. 先掌握 Python 与 API 调用。
2. 再掌握 Prompt、Function Calling、RAG。
3. 然后学习 LangChain / LangGraph / LlamaIndex 等框架。
4. 最后做一个可演示、可部署、有日志的 Agent 项目。

### 项目方向

- 可调用天气、搜索、文件工具的个人助手 Agent
- 企业知识库 + 工具调用 Agent
- 办公自动化 Agent：会议纪要、待办提取、日程查询
- 研发效能 Agent：代码检查、告警归因、接口文档生成

## 2. RAG / 知识库 / 工作流工程

### 方向说明

该方向关注如何让 LLM 使用企业文档、业务知识和结构化数据，减少幻觉并提升回答准确率。

### 典型岗位

- JD-06 腾娱-后台开发（王者荣耀子公司）
- JD-13 AI应用开发工程师（社招）
- JD-19 AI应用工程师
- JD-26 AI Agent开发工程师

### 核心能力

- 文档解析与数据清洗
- Chunking / 分块策略
- Embedding / 向量化
- 向量数据库：Milvus、FAISS、PGVector、Qdrant、Elasticsearch
- 混合检索与 Rerank
- 检索效果评估
- RAG 与 Agent 工作流结合

### 学习重点

1. 掌握 RAG 全链路，而不是只会调用向量库。
2. 理解召回率、准确率、幻觉、上下文窗口和重排的关系。
3. 学会为 RAG 加日志、评测和调优闭环。

### 项目方向

- 个人知识库问答系统
- 企业制度 / 产品文档问答助手
- 带引用来源和召回评估的 RAG 系统
- RAG + 工具调用 + 工作流的业务 Agent

## 3. AI 应用后端开发

### 方向说明

该方向适合有后端基础的人，核心是把 LLM、RAG、Agent 能力封装为稳定服务，并支撑业务系统使用。

### 典型岗位

- JD-06 腾娱-后台开发（王者荣耀子公司）
- JD-08 AI应用后台开发工程师 | 腾讯
- JD-13 AI应用开发工程师（社招）
- JD-22 AI Agent开发工程师
- JD-23 AI后端工程师（OPPO小布Agent）
- JD-28 AI应用工程师（Anker）

### 核心能力

- Python / Java / Go / TypeScript 后端开发
- FastAPI / Flask / Django / Spring Boot / Express / Koa
- RESTful API 设计
- MySQL / PostgreSQL / Redis / MongoDB
- 消息队列、缓存、重试、限流
- 流式响应
- 日志、监控、错误恢复
- Docker / Linux / K8s

### 学习重点

1. 用后端服务承载 LLM 应用，而不是只写 Notebook Demo。
2. 掌握流式输出、异步任务、错误处理、缓存和成本控制。
3. 理解高并发、高可用、可观测性在 AI 应用中的意义。

### 项目方向

- FastAPI + LLM API 聊天服务
- FastAPI + RAG 知识库后端
- 带 Redis 缓存和日志的 Agent 服务
- 支持流式响应和任务队列的 AI 后端系统

## 4. 模型训练 / 微调 / 算法工程

### 方向说明

该方向更偏算法与模型工程，需要深度学习、模型部署、微调和推理优化能力。对于目标是 AI Agent 应用开发的人，它属于进阶或加分方向，不一定最先学习。

### 典型岗位

- JD-03 AI/Agent模型工程师
- JD-07 AI研发工程师（心悦互娱）
- JD-11 AI算法工程师
- JD-18 Agent开发工程师

### 核心能力

- PyTorch / TensorFlow
- 深度学习基础：CNN、RNN、GAN、迁移学习
- NLP / 多模态 / CLIP
- 模型训练、微调、推理部署
- CUDA / C++ / Linux
- 模型评测与调参

### 学习重点

1. 先理解 LLM 应用开发与模型训练的边界。
2. 如果求职目标是应用开发，先把 RAG / Agent / 后端做扎实。
3. 如果目标是算法岗，再系统补深度学习、模型训练和推理优化。

### 项目方向

- 小型文本分类或图像分类模型训练
- 开源模型本地推理部署
- LoRA 微调实验
- 模型评测报告

## 5. AI 产品 / 运营 / 业务落地

### 方向说明

该方向强调业务流程拆解、Prompt 策略、AI 工具引入、指标监控和跨团队推动能力。它不一定要求深度编码，但需要能把 AI 能力落到真实业务。

### 典型岗位

- JD-04 大模型运营专家（Agent）| 快手
- JD-10 AI产品经理
- JD-24 AI战略储备/管培生（精英人才计划）

### 核心能力

- 业务流程拆解
- Prompt / 指令工程
- 知识库建设
- AI 工具选型
- 指标监控：准确率、召回率、任务成功率、成本
- 跨团队协作
- 方案表达与文档能力

### 学习重点

1. 学会把业务问题改写成 Agent 任务。
2. 学会定义角色、工具、输入输出和验收指标。
3. 学会用 Demo、流程图、Prompt 和数据指标说明方案价值。

### 项目方向

- 广告审核规则 Agent 方案
- 办公流程自动化 Agent 原型
- 智慧停车业务拆解 + Prompt + 工具流程
- AI 产品 PRD + 可运行 Demo

## 6. 行业场景型 AI 应用

### 方向说明

行业场景型岗位通常要求“技术 + 业务理解”。技术栈可能类似，但评价重点是能否把 Agent 真正嵌入业务流程。

### 典型场景与岗位

| 场景 | 代表岗位 | 重点能力 |
|---|---|---|
| 教育 | JD-05 | 教学业务拆解、知识库、学习助手、任务规划 |
| 游戏 | JD-06、JD-07、JD-27 | 游戏工具链、AIGC、美术流程、后端工程 |
| 办公 | JD-20 | 多 Agent、会议纪要、待办提取、企业文档检索 |
| 餐饮 / 新零售 | JD-14、JD-29 | 智能选品、运营分析、智能客服、研发效能 |
| 广告审核 | JD-04 | 规则知识库、风险识别、准确率 / 召回率监控 |
| 智慧停车 | JD-24 | 多 Agent 角色拆解、业务闭环、AI 工具实操 |
| 网络优化 / 投诉分析 | JD-30 | 业务 API、知识库、可观测性、LLMOps |

### 学习重点

1. 每做一个项目，都要明确业务场景，而不是只堆技术名词。
2. 项目 README 中要写清楚用户是谁、流程是什么、Agent 解决了什么问题。
3. 面试表达时要突出“需求拆解 → 技术方案 → 效果验证 → 迭代优化”。
