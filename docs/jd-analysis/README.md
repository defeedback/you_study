# AI Agent 岗位 JD 分析总览

## 资料来源

- 原始 JD 合集：[../../岗位JD.md](../../岗位JD.md)
- 分析范围：30 份 AI / Agent 相关岗位 JD
- 使用目标：从岗位需求中提炼 AI Agent 应用开发所需的技术栈、能力地图和项目实践方向

## 文档索引

1. [标准化岗位清单](01-jd-cleaned.md)：将 30 个岗位统一为 `JD-01` 到 `JD-30`，便于横向对比。
2. [岗位方向分类](02-role-taxonomy.md)：按学习和求职方向重新组织岗位，而不是只按原始批次查看。
3. [技术栈矩阵](03-tech-stack-matrix.md)：汇总高频技术、代表岗位、学习优先级和实践项目。
4. [能力地图](04-capability-map.md)：把 JD 技术要求转换为可学习、可练习、可做项目的能力模块。

## 使用方式

- 想看原始岗位内容：阅读 [../../岗位JD.md](../../岗位JD.md)。
- 想快速比较 30 个岗位：阅读 [01-jd-cleaned.md](01-jd-cleaned.md)。
- 想知道岗位可以分成哪些方向：阅读 [02-role-taxonomy.md](02-role-taxonomy.md)。
- 想知道应该学哪些技术：阅读 [03-tech-stack-matrix.md](03-tech-stack-matrix.md)。
- 想把技术转成学习任务和项目：阅读 [04-capability-map.md](04-capability-map.md)。

## 整理原则

- 保留原始 JD 信息，不随意删除岗位职责、任职要求和加分项。
- 将同义字段统一为便于学习分析的结构，例如“工作内容”归入“岗位职责”，“优先项”归入“加分项”。
- 将原始岗位信息与分析结论分开，避免为了学习路线而改变 JD 原意。
- 学习路线应在建立用户画像后再个性化制定，本目录先提供通用能力框架。
