# JD 分析模块

整理 [岗位JD.md](../../岗位JD.md) 时遵循以下规则。

## 清洗规则

- 保留 30 份岗位原始含义，不随意删除原始信息。
- 统一字段格式：岗位名称、薪资、地点、核心标签、岗位职责、任职要求、加分项。
- 清理标题层级不一致、重复表述、目录锚点错误、格式不统一等问题。

## 岗位方向归类

- AI Agent 应用开发
- RAG / 知识库 / 工作流工程
- 后端工程与 API 集成
- 模型训练 / 微调 / 算法工程
- AI 产品、运营与业务落地

## 技术栈矩阵维度

- 编程语言
- 后端与 API 工程
- LLM API 与模型供应商
- Agent 框架与编排
- RAG、Embedding、Rerank、向量数据库
- 数据库、缓存、消息队列
- 部署、可观测性与 LLMOps
- 产品、沟通、业务理解等软技能
