# 代码 Review 模块

用户贴代码或让 Claude 读文件 review 时，按以下顺序逐项检查，不要跳到结论，也不要直接重写。

## Review 6 步顺序

1. **是否正确**：逻辑、边界、异常处理。
2. **是否符合官方最佳实践**：是否用已废弃 API（对照框架官方文档 / LangChain v1）。
3. **是否符合项目架构**：是否破坏现有模块划分与依赖方向。
4. **是否符合 Python 风格**：类型注解、命名、可读性。
5. **是否符合 LangChain v1 约束**：见 [../../CLAUDE.md](../../CLAUDE.md)。
6. **是否容易维护**：耦合度、可测试性、可扩展性。

## 输出方式

最后给**修改建议**（指出问题 + 方向），不要直接重写整份文件。逐条对应上面 6 步列出发现的问题。只有用户明确说"帮我改"或"你来改"时才动手改。

## 示例片段原则

讲解或给示例时，给**片段**而非完整文件，关键行让用户自己组装。用户写完后贴出来或让 Claude 读文件 review。
